01

Core Concepts

Query, Key, Value
三位一体

注意力机制将输入转换为三种不同的表示，各司其职，在信息交换中扮演独特角色。

Q

Query 查询

Q = X · W_Q

"我在寻找什么信息"——每个位置发出查询信号，主动在序列中搜索相关上下文。

K

Key 键

K = X · W_K

"我能提供什么匹配"——定义每个位置的特征标签，与查询计算相关性分数。

V

Value 值

V = X · W_V

"我实际传递的内容"——承载语义信息，根据注意力权重被加权聚合。

02

The Formula

缩放点积注意力

一个简洁而强大的公式，定义了信息如何在序列中流动与聚合。

Scaled Dot-Product Attention

Attention(Q, K, V) = softmax(QKT / √dk) · V

计算 Query 与 Key 的点积相似度，通过缩放因子防止梯度消失，经过 Softmax 归一化后加权聚合 Value。

Design Rationale

为什么分离
Q、K、V？

三个独立投影是成功的关键设计决策。

  • 1

    打破对称性

    "猫注意狗" ≠ "狗注意猫"

  • 2

    信息解耦

    "匹配什么"与"传递什么"分离

  • 3

    增强表达力

    3×d² 可学习参数

  • 4

    跨注意力支持

    编码器-解码器信息流动

Implementation

代码实现

attention.py 
class Attention(nn.Module):
    def __init__(self, d, d_k):
        super().__init__()
        self.W_q = nn.Linear(d, d_k)
        self.W_k = nn.Linear(d, d_k)
        self.W_v = nn.Linear(d, d_k)
        self.scale = d_k ** 0.5

    def forward(self, x):
        Q = self.W_q(x)
        K = self.W_k(x)
        V = self.W_v(x)

        scores = Q @ K.T / self.scale
        attn = F.softmax(scores, -1)
        return attn @ V
